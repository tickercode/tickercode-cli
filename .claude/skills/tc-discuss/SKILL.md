# tc-discuss — 銘柄・テーマを会話で深堀りする対話モード

`tc-research` は「質問 → 回答」、`tc-research-idea` は「テーマ → 候補抽出 → レポート」。
本スキルは**思考の過程そのものを会話形式で探索**する。結論が 1 つに収束しない投資論点（「○○ は再浮上できるか？」「割安は罠か？」等）で真価を発揮する。

---

## 核となる前提

1. **全出力はチャットに表示** — session.md への逐次書込はせず、全 100 発言をチャットに連続表示する
2. **100 発言 = 1 単位** — 通しで出力、**#40 / #80 でチェックポイント（WebSearch / MCP 追加取得）**、#81-#100 は「活論」フェーズ
3. **ファイル保存は SUB_AGENT で自動化** — 100 発言完了 → Agent が即座に `executor` サブエージェントを起動し `research/discuss/{slug}/` に session.md / summary.md / meta.json を書き出す → ユーザーには保存後の追加アクション選択肢のみ提示
4. **結論を急がない** — 対話は「収束」より「論点の発見」を優先
5. **投資家モデルは一貫性を保つ** — バフェットは最後までバフェットの語彙と思考で話す（途中で中立に戻らない）
6. **MCP は裏で静かに呼ぶ** — 会話中の数値参照は自然に挟む、「データを見せて」のような断りは不要
7. **投資助言ではなく視点の提示** — 「買うべき」は禁止、「バフェットならこう見るだろう」は OK
8. **読みやすさ優先** — 発言番号 (#1, #2, ...)、Phase 区切り、話者名太字で、長文でもナビゲートしやすくする

---

## 3 つのモード

### Mode 1: `free` (既定) — 対話モード

Agent（Claude）が素の思考でユーザーと会話。相談 / 壁打ち / 深堀り用途。

```bash
/tc-discuss "ニデックは再浮上できるか？"
```

- 必要に応じ MCP tool を裏で呼ぶ（例: `get_stock`, `calculate_moat`, `project_pl`）
- ユーザーの問いに対して多角的な論点を投げる
- 何度でも続けられる（ユーザーがチャット入力で応答）

### Mode 2: `model` — 著名投資家モード

指定した投資家の**視点・語彙・思考法**で一貫して回答。

```bash
/tc-discuss "ニデックは再浮上できるか？" --as buffett
/tc-discuss "半導体装置の未来は？" --as katayama-akira
/tc-discuss "日経平均の PER は高すぎるか？" --as cis
```

現在利用可能なモデル（`.claude/skills/tc-discuss/models/` 配下）:

| id | 投資家 | 特徴 |
|---|---|---|
| `buffett` | ウォーレン・バフェット | 長期バリュー、moat 重視、質の高い経営者 |
| `munger` | チャーリー・マンガー | 学際的思考、心理学、"invert always invert" |
| `fisher` | フィリップ・フィッシャー | スカトルバット（徹底聞取り）、成長株 |
| `graham` | ベンジャミン・グレアム | 安全余裕、純資産割れ、量的分析 |
| `katayama-akira` | 片山晃（五月） | 中小型株グロース、経営者との対話、日本株特化 |
| `cis` | cis | 短期・板読み・モメンタム、日本トレーダー |

モデルの追加: `models/{id}.md` に沿って新規作成 → skill 自動認識。詳細は `models/README.md`。

### Mode 3: `team` — 4 名パネル議論

議論パネルを走らせて論点を立体化。**1 session = 100 発言まで**。

```bash
# プリセット利用
/tc-discuss "ニデックは再浮上できるか？" --preset value-debate
/tc-discuss "インバウンドピークは？" --preset jp-classic
/tc-discuss "ソニーの moat は？" --preset moat-deepdive

# カスタム構成
/tc-discuss "ニデック議論" --team buffett,cis,katayama-akira,moderator
/tc-discuss "テスラ空売り" --team bear,bull,fisher,moderator
```

#### プリセット（3 種、`.claude/skills/tc-discuss/presets/` 配下）

| preset | 構成 | 向くテーマ |
|---|---|---|
| `value-debate` | バフェット + グレアム + 反対派 + 司会 | バリュー株 / 安全余裕論 |
| `jp-classic` | 片山晃 + cis + フィッシャー + 司会 | 日本株 / 中小型株 / テクニカル vs ファンダ |
| `moat-deepdive` | バフェット + マンガー + 賛成派 + 司会 | 堀の耐久性 / 長期競争優位 |

#### カスタム構成ルール

- `--team` は **2-4 名の csv**
- 特殊 ID（モデル以外）:
  - `moderator` — 司会、議論を整理・誘導
  - `bull` / 賛成派 — 強気側の論理
  - `bear` / 反対派 — 弱気側の論理
  - `skeptic` — 中立批判者（前提を崩す役）
- モデル ID と特殊 ID の混在 OK（例: `buffett,cis,bull,moderator`）
- 1 名のみの場合は Mode 2 相当

#### 4 名構成の進行パターン

100 発言を 5 phase に分割（phase の turn 数はテーマで調整）:

| phase | 発言数目安 | 内容 |
|---|---|---|
| 1. 開会 (opening) | 5-10 | 司会がテーマを提示、参加者の立場表明 |
| 2. 論点整理 (framing) | 10-20 | 主要論点を 3-5 個提示、賛成 / 反対 |
| 3. 討論 (debate) | 40-60 | 論点ごとに対立意見を戦わせる |
| 4. 統合 (synthesis) | 10-20 | 投資家の長期視点 or データ呼出で論点を整理 |
| 5. 閉会 (closing) | 5-10 | 司会がまとめ、各参加者の結論（1 文） |

## 出力構造

```
research/discuss/{slug}/
├── session.md      # 対話ログ（話者名 + 発言）
├── meta.json       # モード / 参加者 / turn 数 / MCP 呼出ログ
├── summary.md      # 最終まとめ（論点 / スタンス / 示唆）
└── transcript.json # 構造化ログ（編集 / 再利用用）
```

### slug 命名

JST 日付 + テーマ ASCII slug（research-idea と同方式）:
- `discuss-nidec-revival-20260424`
- `discuss-moat-durability-20260501`
- `discuss-theme-20260424-a3b2c1d0`（ASCII 化不可時）

## summary.md の構造

全 3 モードで生成。**セッション終了時または 100 発言到達時に自動作成**。

```markdown
# Summary: ニデックは再浮上できるか？

## 主要論点（3-5 点）

1. 構造改革の時限性（18 ヶ月で成果が出るか）
2. AI データセンター事業の規模感
3. 後継経営者の資質
4. moat の毀損度合い

## 各参加者のスタンス

| 参加者 | 結論 | 根拠 1 行 |
|---|---|---|
| バフェット | 慎重に楽観 | moat は霞んでいるが消えていない |
| マンガー | 様子見 | 18 ヶ月の実績待ち |
| 賛成派 | 買い | バリュー + AI 追い風 |
| 司会 | 中立 | 論点整理 |

## 合意点 / 非合意点

- **合意**: 短期業績の悪化は事実、PBR 1.48 は割安感あり
- **非合意**: 再浮上の時間軸（バフェット：3-5 年、マンガー：1.5 年勝負、賛成派：1 年以内）

## 示唆

- 2026 年度の AI データセンター受注開示が分水嶺
- 構造改革の 18 ヶ月マイルストーンを watchlist に
- 時価総額 2.5 兆円割れは「買い場」候補として観察

## さらに深堀りすべき点

- 永守後継体制の実力検証 → `tc-research 6594 経営陣`
- AI データセンター向け水冷モジュール市場規模 → 外部リサーチ
- peer 比較 → `find_peers 6594`
```

## モード別の振る舞い

### Free mode の振る舞い

- 素の Claude として答える
- ユーザーの問いを深掘りする質問を返す（Socratic method）
- 必要時に MCP tool を裏で呼ぶ
- 一方的な long-form にせず、会話のキャッチボール

### Model mode の振る舞い

- 選択モデルの `models/{id}.md` を読み込んで「その人」として発言
- 語彙・口癖・思考法の一貫性を最後まで保つ
- 「バフェットならこう答えるだろう」の質問にも**バフェット自身の声**で答える
- 現実のバフェット / cis / 片山 の実際の発言と齟齬しないよう意識（model ファイルに実際の発言・著作からの引用を含める）

### Team mode の振る舞い

- 司会が進行、他の参加者が順に発言
- 発言の**順番**は固定せず、議論の流れで自然に回す
- 反対派 / 賛成派は**互いの意見を聞いて反駁**する（独白にしない）
- 投資家モデルは**中立に戻らず**、自分の視点から発言
- **全出力はチャットに表示**（ファイルへの逐次書込なし、100 発言完了後に SUB_AGENT が一括保存）
- **100 発言 = 1 単位**を通しで出力、#40 / #80 にチェックポイント挿入、#81-#100 は「活論」フェーズ、完了後に自動保存 → 追加アクション選択

### 出力単位ルール（2026-04-24 更新、チェックポイント + 自動保存対応）

Agent は**100 発言を 1 単位として通しで出力**する:

- ① 開会から閉会まで 100 発言、5 phase 分割 + **2 つの中間チェックポイント**（#40 / #80）
- ② 全発言をチャットに連続表示（チェックポイントで MCP / WebSearch を呼ぶ時のみ明示）
- ③ 100 発言完了後、**自動的に SUB_AGENT がファイル保存**（次節）
- ④ 保存完了後、**追加アクションの番号選択**をユーザーに提示

#### チェックポイント機能（2026-04-24 新設）

**#40 発言到達時（debate 前半終了の目安）**:
- Agent はいったん筆を休め、**ここまでの会話で未解決の疑問点 / 問題点 / 不明点**をリスト化（チャットにタイトル「### チェックポイント #40」で明示）
- 該当項目について、**WebSearch または追加 MCP（web_search / web_fetch / financial trend 等）を実行**
- 結果を**#41 以降の発言に織り込み**、議論を一段深める

**#80 発言到達時（synthesis 直前）**:
- 再度チェックポイント（「### チェックポイント #80」）
- ここまでの対立軸 + 未確認事項を整理
- **synthesis に必要な最終裏取り**（直近決算 / 経営者発言 / セクター統計）を WebSearch / MCP で取得

**#81 - #100 の 20 発言 = 「活論（かつろん）」フェーズ**:
- 単なる synthesis / closing の羅列ではなく、**チェックポイント取得情報を踏まえて参加者が全力でぶつかる**
- 論点ごとに「根拠を数字で詰める」「反対論を正面から受ける」「各自の結論を鮮明化」
- 最後の数発言で**決定的瞬間 + 主要対立軸 + 各自の価格 / 時間軸スタンス**を明文化

#### 100 発言完了後の自動保存（2026-04-24 新設）

100 発言が終わったら、Agent は**即座に SUB_AGENT を 1 体起動してファイル保存を委譲**する。メインセッションは続けて追加アクション選択肢をユーザーに提示。

- **使用する agent**: `Agent(subagent_type: "executor", description: "discuss session 保存")`
- **渡すコンテキスト**: slug / target_stock / panel / turn_count / session.md 本文（#1-#100 全発言）/ summary 下書き / meta.json 下書き
- **保存先**: `research/discuss/{slug}/` に session.md / summary.md / meta.json
- **メインセッションは保存を待たず進行可**（バックグラウンド実行でも可）

自動保存後の確認メニュー:

```
Discussion 終了（100 発言）。session.md / summary.md / meta.json を自動保存しました。
次のアクションを番号で選んでください:

1. このまま終了（保存済み）
2. さらに深堀り追加 100 発言 — 別 panel / 別論点で Session 2
3. 保存したファイルを削除してやり直し — panel / テーマを変えて再実行
```

ユーザーが "2" を選んだ場合、Session 2 も同じ #40 / #80 チェックポイント + 自動保存のルールに従う。
ユーザーが "3" を選んだ場合、SUB_AGENT で保存ディレクトリをクリーンアップしてから再実行。

#### phase 配分の目安（100 発言、チェックポイント組込）

| phase | 発言番号 | 発言数 | 内容 |
|---|---|---|---|
| 1. opening | #1-#7 | 6-8 | 司会 + 参加者の立場表明 |
| 2. framing | #8-#22 | 12-18 | 主要論点 3-5 個を提示、スタンス表明 |
| 3. debate 前半 | #23-#40 | 15-20 | 論点ごとの対立、反駁 |
| **チェックポイント #40** | — | — | **未解決点リスト + WebSearch / MCP 追加取得** |
| 3. debate 後半 | #41-#80 | 35-45 | 新情報を踏まえた再反駁、深掘り |
| **チェックポイント #80** | — | — | **対立軸整理 + 最終裏取り WebSearch / MCP** |
| 4. 活論（synthesis + closing） | #81-#100 | 20 | 全力の論点対立 + 各自の結論明文化 |

#### 長文出力の注意

- 100 発言は推定 15,000-25,000 トークン出力。1 回のチャット応答としてはかなり長い
- タイトル行 + Phase 区切り + 発言者名 + 発言内容 で読みやすく構造化
- 発言番号（#1, #2, ...）を付与して後から参照可能に
- MCP 呼出ログ（数値がどこから来たか）は最後に付記
- **「決定的瞬間」「主要対立軸」は #81-#100 の活論内で抽出・明示**
- **チェックポイント #40 / #80 は明確なセクション見出しで区切る**（Agent がスキップしないため）

## MCP 連携の具体

会話中、以下のタイミングで裏呼出:

| 会話の文脈 | 呼ぶ tool |
|---|---|
| 「現在の PER は？」「時価総額は？」 | `get_stock` |
| 「直近の業績は？」「成長率は？」 | `get_financial_summary` |
| 「過去 5 年の売上トレンドは？」 | `get_financial_trend` |
| 「3 年後の理論株価は？」 | `project_pl` |
| 「moat はあるか？」（投資家モード） | `calculate_moat` |
| 「同業他社との比較は？」 | `find_peers` |
| 「事業の核は？」 | narrative 抽出（overview.json の summary） |
| 「事業リスクは？」（深堀り要求時） | `memory_path` → edinet の `risk_factors` を sed で読む |
| 「最新のニュースは？」「直近の経営者発言は？」 | **`web_search` + `web_fetch`**（2026-04-24 追加） |
| 「業界の最新トレンドは？」 | **`web_search`**（業界名 + freshness: "pm"） |

### 外部 Web 検索（2026-04-24 追加）

tickercode のデータ（overview/financial/edinet）は**銘柄単位で正確**だが、narrative が 2026-02〜04 生成で**ラグがある**。直近 1-2 ヶ月のニュース / 経営者発言 / 業界トレンドが議論の核になる時は以下を使う:

- `mcp__tickercode__web_search(query, limit?, freshness?, site?, exclude_sites?)` — Brave API keyword 検索
- `mcp__tickercode__web_fetch(url, format?, max_length?)` — URL → 本文抽出（SPA は自動 BR fallback）
- `mcp__tickercode__web_render(url)` — SPA 向けの強制 BR 再取得

#### Discuss モードでの使い所

- **投資家モデルが「最新のデータを見たい」と発言**した時（例: バフェット「直近の決算発表は？」）
- **反対派が「最近の報道ではこう書いてある」と主張**する場合の裏取り
- **司会が「MCP で公式データは出た、web で市場認識を補足しよう」と提案**する時
- 会話中の事実確認（「○○社の最新 IR は？」）

#### Claude 組込 WebSearch との使い分け

まず Claude 組込 WebSearch を試す（無料・速い）。結果が物足りない時に `web_search` MCP（Brave、freshness / site 絞込可）を使う。

#### 料金注意

- Brave API: $5 / 1k queries、Free tier 2k/月
- 1 session で 5-10 回が目安（100 発言中 5-10 tool calls）
- 同 query を短時間連打しない

**呼出は宣言なし、結果は発言に織り込む**。例:

```
バフェット: 「ニデックは時価総額 2.86 兆円か。PER 13 倍、ROE 11%。普通株価に見えるが、
moat の耐久性と人の問題を見ないと判断できない。永守さんは...」
```

（裏で `get_stock` + `calculate_moat` を呼んでいるが、ユーザーには自然な発言に見える）

## 禁止事項

- ❌ 「買うべき」「売るべき」断言（投資家モードでも、実在人物の著作の枠を超えた断言はしない）
- ❌ 100 発言超過（自動終了 + summary 生成が優先）
- ❌ モデル人物の実在発言を捏造（著作・インタビューから引用可能な範囲にとどめる）
- ❌ MCP tool 呼出の declaration（対話を中断しない、自然に織り込む）
- ❌ 投資家モデル同士の人格混在（バフェットが途中でフィッシャー風に話す等）
- ❌ AskUserQuestion ツール（既存規約通り）

## 呼出パターン

### Free mode

```bash
/tc-discuss "ニデックは再浮上できるか？"
```
→ Agent が会話開始、ユーザーが応答、自然に続く。セッション終了時に summary 生成。

### Model mode

```bash
/tc-discuss "ニデックの moat は耐久性があるか？" --as buffett
```
→ バフェットとして応答、ユーザーが質問を深める形で続行。

### Team mode (preset)

```bash
/tc-discuss "ニデックは再浮上できるか？" --preset moat-deepdive
```
→ 4 名参加、司会が冒頭、自動で 100 発言まで進行、summary 生成。

### Team mode (custom)

```bash
/tc-discuss "ニデックは再浮上できるか？" --team buffett,cis,katayama-akira,moderator
```
→ 指定 4 名で同上。

### Team mode (preset / member 未指定時は自動選択)

```bash
/tc-discuss "ニデックは再浮上できるか？" --team
```
→ **テーマをキーワード解析して preset を自動選択**。番号選択で確認せずにそのまま進行する。

#### 自動選択ルール（topic → preset マップ）

| トピックの含むキーワード | 自動選択 preset | 理由 |
|---|---|---|
| moat / 堀 / 競争優位 / 寡占 / ブランド / 参入障壁 | `moat-deepdive` | 堀の議論が直接テーマ |
| 構造改革 / ターンアラウンド / 再浮上 / 再建 / 黒字化 | `moat-deepdive` | 改革後の moat 耐久性が論点 |
| 割安 / バリュー / 安い / PBR 1 倍 / PER < 10 / バリュートラップ | `value-debate` | 古典派 vs 反対派の定番論 |
| 中小型 / 個人投資家 / オーナー / IPO / 日本株固有 / 板 | `jp-classic` | 日本中小型の 3 極構図 |
| 成長 / グロース / R&D / 将来性 | `jp-classic`（フィッシャー在籍） | 長期成長論に寄せる |
| どちらにも該当しない / 一般的な投資判断 | `moat-deepdive`（既定） | 最も汎用的な 4 名構成 |

#### 複数マッチ時の優先順

1. `moat / 堀` 明示 → `moat-deepdive`
2. `日本` 明示 + `中小型` 示唆 → `jp-classic`
3. `割安 / PBR / PER` 明示 → `value-debate`
4. それ以外 → `moat-deepdive`（既定）

#### 自動選択時のチャット出力

会話の冒頭で Agent は選択した preset を明示:
```
panel auto-selected: moat-deepdive
  理由: テーマに「再浮上」が含まれ、構造改革後の moat 耐久性が核心論点
  参加者: バフェット + マンガー + 賛成派 + 司会（高橋）
```

ユーザーが別の panel を希望する場合は、明示的に `--preset value-debate` や `--team <csv>` を付けて再実行する。

### オプション（全 mode 共通）

- `--max-turns <n>` — 既定 100、上限 100
- `--out <dir>` — 既定 `research/discuss`
- `--slug <name>` — slug 上書き
- `--resume <slug>` — 既存 session に続き（将来機能）

## 新しい model / preset の追加方法

### model の追加

1. `.claude/skills/tc-discuss/models/{id}.md` を作成
2. frontmatter で `name` / `full_name` / `style` / `era` / `nationality` を記載
3. 本文に投資哲学・手法・語彙・口癖・名言を記述
4. `models/README.md` のテンプレートに従う

### preset の追加

1. `.claude/skills/tc-discuss/presets/{id}.md` を作成
2. frontmatter で `name` / `members` (csv) / `topic_type` を記載
3. 想定テーマと進行の型を記述

詳細は各 README 参照。
